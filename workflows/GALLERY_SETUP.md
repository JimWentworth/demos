# Auto-Update Gallery Index

This GitHub Action automatically regenerates your `index.html` gallery whenever you push new HTML files to the repository.

## Setup Instructions

1. **Create the workflow directory** in your demos repository:
   ```bash
   mkdir -p .github/workflows
   ```

2. **Add the workflow file**:
   - Copy `update-gallery.yml` to `.github/workflows/update-gallery.yml`

3. **Commit and push**:
   ```bash
   git add .github/workflows/update-gallery.yml
   git commit -m "Add auto-update gallery workflow"
   git push
   ```

## How It Works

- **Automatic**: Runs every time you push to the `main` branch
- **Manual**: Can also be triggered manually from the Actions tab in GitHub
- **Smart**: Only commits changes if the index.html actually changed
- **Fast**: Takes about 30 seconds to run

## What It Does

1. Scans all `.html` files in the repository (except `index.html` itself)
2. Extracts the `<title>` tag from each file for better descriptions
3. Groups files by directory/category
4. Regenerates the complete `index.html` gallery
5. Commits and pushes the updated index if anything changed

## Adding New Projects

Just add your new `.html` file anywhere in the repository and push:

```bash
git add my-new-demo.html
git commit -m "Add new demo"
git push
```

The workflow will automatically:
- Detect the new file
- Add it to the appropriate category
- Update the gallery index
- Push the changes back to the repository

If the workflow isn't running:
