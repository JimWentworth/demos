Purpose

Build practical intuition (and safer habits)

Treat generative AI as a system that produces plausible outputs from patterns and context—not as an authority. We’ll focus on asking better questions, verifying key claims, and choosing tools that match your data sensitivity.

One sentence that keeps you out of trouble
Before you trust an output, ask: “What evidence would convince me this is correct?”

You’ll see how prompts steer outcomes, why “where info comes from” changes reliability, and how detailed specs can dramatically tighten results.

Understanding Neural Networks: A Useful Comparison (and Its Limits)

Comparing AI neural networks to human neural networks can help with intuition: many small influences combine into a final outcome. But modern language models do not “understand” like humans—they generate likely continuations based on patterns and the context you provide.

Core idea

Connections steer outputs

An LLM uses layers of weighted connections to map context to probable next tokens (pieces of words). Those weights were learned from training data.

Where bias enters
Training data reflects uneven representation and historical patterns. Models can inherit those patterns and present them with a confident tone unless you counterbalance with constraints and verification.
Key limit

Prediction is not perception

Humans rely on grounded experience and situational awareness. A model relies on text (or other digital input) plus learned correlations. When context is missing, it may fill gaps with plausible content.

Practical implication
Better prompting is often “context provisioning”: audience, goal, constraints, examples, and what success looks like.
Activity

“Mary had a little …” and why context controls meaning

Many people expect “lamb.” The model expects that too—until you push it in a different direction. This is a fast way to show how context selects a pathway.

Copilot activity (copy/paste)
Copy/paste prompt — Context steering 
Complete the phrase “Mary had a little …” in four different ways:
1) the most common continuation,
2) a continuation suitable for a university context,
3) a continuation that changes the setting (not a farm),
4) a continuation that subtly shifts the meaning of “little”.

After each completion, briefly explain what words in the prompt shaped your choice.
Reality check: the model’s explanation can be a post-hoc story. Treat it as a hypothesis, not proof.

Where the Information Is Coming From: No Retrieval, Internet, Grounded (RAG)

“Is it reliable?” depends on how the answer was produced. A model might respond from internal training patterns, it might retrieve from the internet, or it might be grounded in a trusted set of documents (RAG: Retrieval-Augmented Generation).

Stage 1

No retrieval

Good for drafting and brainstorming; risky for highly specific facts, policies, or anything that changes over time.

Prompt example
Copy/paste — Model-only response 
Explain “retrieval-augmented generation (RAG)” to a mixed university audience in ~150 words.
Then list two things you are unsure about and how you would verify them.
Stage 2

Internet-supported

Better for “what’s current,” but only if sources are credible and the tool provides citations.

Prompt example
Copy/paste — Internet-supported response 
Using current web sources, summarize what “RAG” means in higher education and why it matters.
Include citations and state how you judged source reliability.
Stage 3

Grounded (RAG)

Most reliable for institutional guidance because you can trace claims back to a known document set.

Prompt example
Copy/paste — Grounded-only 
Answer using ONLY the provided documents:
“What guidance is given about what data may or may not be entered into AI tools?”
Cite the exact lines/sections used. If not found, say “Not found in the provided materials.”
Takeaway

Grounding doesn’t guarantee truth—it makes answers checkable

The strongest habit to build is not “trust the model,” but “require evidence when it matters.”

Guardrails & Specification: The “Talented Intern” Test

If you ask a talented but untrained intern to “build a workshop website,” you’ll get something—but not necessarily what you meant. AI is similar: detailed specifications reduce ambiguity and give you more control over the outcome.

Why guardrails

Boundaries reduce harm (and rework)

Guardrails are most important when stakes are high: protected data, grading decisions, personnel matters, legal/policy interpretation.

Specification in plain language
A good spec names the audience and purpose, sets constraints (format, length, accessibility), provides examples, and explains what “done” looks like. If you can’t describe success criteria, you can’t evaluate the output.
Claude demo

Detailed spec prompt

Run a vague prompt and a detailed spec back-to-back. The contrast is a great teaching moment.

Copy/paste — Website specification (Claude-friendly) 
You are a careful web developer building an accessible, single-file workshop website for University of Illinois faculty/staff/students.

Hard requirements:
- One self-contained HTML file with embedded CSS + JavaScript (no external libraries).
- Accessible navigation using tabs, keyboard-friendly.
- Use cards and native accordions (details/summary).
- At most one embedded video per page.
- Provide copy/paste prompt examples.
- Prioritize Title II/WCAG-friendly patterns: skip link, focus states, semantic headings, readable contrast.

Content requirements:
- Neural networks + context + bias.
- RAG stages: no retrieval, internet-supported, grounded.
- Guardrails + specification.
- Multimodal “bits are bits” (NotebookLM).
- Campus tools (link to https://genai.illinois.edu/ai-apps/ and mention Illinois Chat).
- Learn More page with feedback link and contact.

Output: Return the full HTML file only.
Pushback question for the room: “What would count as evidence this output meets the spec?”

Bits Are Bits: Multimodal AI (Voice, Documents, and More)

Once something is digitized, it becomes “bits”—text, images, audio, video, and data. That’s why AI can work across modes: you can speak to a tool, upload readings, and generate summaries or study aids. The same reliability rule holds: constrain the task and require evidence.

NotebookLM

Document-grounded workflows

NotebookLM is a useful example: it’s designed to work from documents you provide, which fits academic norms (quote sources; separate summary from interpretation).

Evidence-first prompt
Copy/paste — Evidence-first summary 
Summarize the provided materials for a university audience in 6–8 sentences.
Include 3 direct quotes (or precise references) from the source material.
Separate “What the source says” from “My interpretation.”
Caution

Multimodal ≠ more correct

Richer inputs can reduce ambiguity, but can also introduce new errors (misheard audio, missing context, out-of-date documents).

What to require in higher-stakes work
Ask for quotes/citations to the provided materials and require at least one independent verification step for any claim that matters.

Risk lens

Don’t outsource your risk assessment

Use AI for drafting and ideation, but keep humans responsible for decisions—especially when outputs could affect students, employees, or policy compliance.

A quick “stoplight” heuristic
Green: low-stakes drafting/brainstorming. Yellow: summaries of your documents (require quotes). Red: protected data and high-stakes decisions.
